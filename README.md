# Grocery Store Management System (SQL Project)

## Overview
This project is a SQL-based Grocery Store Management System designed to manage and analyze retail store operations. It includes database creation, table relationships, and SQL queries to generate business insights from customers, products, suppliers, employees, orders, and sales data.

## Objectives
- Design a relational database for a grocery store.
- Manage customers, products, suppliers, employees, and orders.
- Analyze sales performance and customer behavior.
- Generate business insights using SQL queries.

## Database Schema
The project contains the following tables:

1. Supplier
2. Categories
3. Employees
4. Customers
5. Products
6. Orders
7. Order_Details

## Key Features
- Database creation and normalization.
- Primary Key and Foreign Key constraints.
- Customer purchase analysis.
- Product performance tracking.
- Sales and revenue analysis.
- Supplier contribution analysis.
- Employee order management insights.

## Business Questions Solved

### Customer Insights
- Unique customers who placed orders.
- Customers with the highest number of orders.
- Total and average purchase value per customer.
- Top customers by revenue.

### Product Analysis
- Products available in each category.
- Product sales volume.
- Revenue generated by each product.
- Category and supplier performance.

### Sales Analysis
- Total orders placed.
- Average order value.
- Order trends over time.
- Revenue analysis.

## Project Outcome
The project helps identify customer purchasing patterns, top-performing products, supplier contributions, and sales trends, enabling data-driven business decisions and improved operational efficiency.
